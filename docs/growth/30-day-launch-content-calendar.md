# 30-Day Launch Content Calendar

## Goal
Build awareness and early social proof for GHCP-MEM through consistent, outcome-focused content that maps to real user pain points.

## Channels
- LinkedIn
- X/Twitter
- GitHub Discussions/Issues updates
- Dev blog (Medium/Hashnode/Personal site)

## Weekly Themes
- Week 1: Problem awareness (context loss, repetitive prompting)
- Week 2: Proof of value (workflow demos, before/after)
- Week 3: Trust and enterprise fit (privacy, no ports, local-first)
- Week 4: Integration and adoption (MCP clients, team rollout)

## Day-by-Day Plan

### Week 1
- Day 1: Launch post — what GHCP-MEM solves in one sentence + GIF.
- Day 2: Short clip — "recover yesterday's decisions in 10 seconds".
- Day 3: Thread — 5 common context-loss pains and fixes.
- Day 4: Technical note — how redaction works (high level).
- Day 5: Tip card — best settings for first-time users.
- Day 6: Community prompt — ask users for biggest context pain.
- Day 7: Weekly recap + top feedback themes.

### Week 2
- Day 8: Before/after story — no memory vs GHCP-MEM workflow.
- Day 9: Demo — `@mem /search` + `/detail` quick retrieval flow.
- Day 10: Demo — export markdown session for PR context.
- Day 11: Mini case — bugfix continuity across sessions.
- Day 12: Integration spotlight — external MCP client setup.
- Day 13: FAQ post — answer top 5 setup questions.
- Day 14: Weekly recap + install milestone check.

### Week 3
- Day 15: Enterprise angle — no sidecar, no open ports.
- Day 16: Security angle — local storage and secret redaction.
- Day 17: Azure angle — subsystem tagging and context snapshots.
- Day 18: Deep dive — retrieval quality and token efficiency.
- Day 19: Reliability note — tests and release quality gate.
- Day 20: User testimonial request post.
- Day 21: Weekly recap + trust highlights.

### Week 4
- Day 22: Team onboarding playbook (5-minute setup).
- Day 23: Demo — workspace/repo scoped retrieval.
- Day 24: Demo — health score and maintenance flow.
- Day 25: Product roadmap post (next 30 days).
- Day 26: Community Q&A highlights.
- Day 27: Case study snippet with measurable outcome.
- Day 28: Weekly recap + CTA for ratings.

### Final 2 Days
- Day 29: Month summary — wins, lessons, next goals.
- Day 30: Strong CTA — install, try one workflow, leave feedback.

## Post Template
- Hook: 1-sentence pain point.
- Proof: short clip/screenshot.
- Outcome: concrete time/risk reduction.
- CTA: install link + one command to try.

## Content KPIs (Weekly)
- Post impressions
- Click-through to Marketplace
- Installs (delta)
- Ratings count (delta)
- Comments/questions count

## Operating Rules
- Keep every post outcome-focused.
- Avoid generic AI claims; show concrete workflows.
- Reuse top-performing formats with different use cases.
