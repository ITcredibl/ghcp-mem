# 90-Day Marketplace Score Plan

## Objective
Move GHCP-MEM from early Marketplace maturity to top-tier trust and adoption signals over 90 days, with measurable checkpoints each month.

## North-Star Metrics
- Installs: >= 2,000 by day 90
- Ratings: >= 25 with average >= 4.5
- Weekly active users (proxy): steady growth in command usage and issue discussions
- Support responsiveness: first maintainer response within 48 hours
- Release discipline: at least 1 quality update every 2 weeks

## Pillars
1. Product Trust
- Keep security posture visible (local-first, no telemetry, redaction details).
- Eliminate secret hygiene risks in repo and examples.
- Publish clear SECURITY.md guidance and disclosure process.

2. Conversion
- Improve Marketplace listing above-the-fold value proposition.
- Keep one short GIF showing user outcome, not architecture.
- Add clear "who this is for" and "why this over alternatives" sections.

3. Distribution
- Ship repeatable social content and practical demos.
- Publish hands-on tutorials and Azure-focused workflows.
- Engage with extension communities and VS Code newsletters.

4. Retention
- Prompt for ratings only after proven value moments.
- Keep onboarding friction low with fast start and troubleshooting docs.
- Maintain predictable release and communication cadence.

## Execution Timeline

### Days 1-14: Trust + Conversion Foundation
- Finalize listing copy refresh and demo assets.
- Publish security + privacy trust section updates.
- Ensure setup docs cover top 10 failure modes.
- Enable respectful in-product rating prompt flow.

Exit Criteria:
- Listing is updated and consistent with README.
- Rating flow shipped and validated.
- 0 unresolved high-severity trust issues in repo docs.

### Days 15-45: Distribution Ramp
- Publish 2 long-form articles and 6 short demo posts.
- Create one repeatable weekly "tip + outcome" content format.
- Track inbound installs after each content drop.
- Run one community AMA/demo session.

Exit Criteria:
- >= 500 installs
- >= 10 ratings
- >= 1 external mention/roundup

### Days 46-90: Scale + Social Proof
- Add 1-2 mini case studies with measurable outcomes.
- Continue biweekly release cadence with clear notes.
- Tighten issue/PR response SLA and public roadmap clarity.
- Re-test Marketplace funnel copy and visuals based on data.

Exit Criteria:
- >= 2,000 installs
- >= 25 ratings, avg >= 4.5
- steady month-over-month growth trend

## Risk Register
- Low visibility despite quality releases
  - Mitigation: increase distribution cadence and partnerships.
- Negative ratings from setup friction
  - Mitigation: strengthen quick-start checks and error docs.
- Growth work displaces engineering quality
  - Mitigation: keep release quality gate (compile + tests) mandatory.

## Definition of Success
A sustainable adoption loop where quality releases, strong trust posture, and consistent distribution produce compounding Marketplace installs, ratings, and retention.
